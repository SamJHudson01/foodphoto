# Council Plan: Hosted Storage + Council Stack Migration

**Scope:** Move FoodPhoto from browser-local IndexedDB source-of-truth to Clerk-authenticated hosted data: R2 for photos, Neon/Postgres via Prisma for metadata and roundups, and tRPC for the API boundary.
**Context:** Current app is a Next.js App Router PWA with one large client component, local IndexedDB entries/photos/roundups, direct `/api/roundup`, and no Clerk/tRPC/Prisma/R2 yet.
**Boundaries:** R2 unlisted public reads are acceptable. Writes/deletes must be server-side. Existing IndexedDB photos should be migrated. Nothing local remains source of truth after migration.
**Council dispatched:** Troy Hunt (security), Martin Fowler (refactoring), Kent C. Dodds (frontend), Matteo Collina (backend/tRPC), Brandur Leach (Postgres/Prisma/Neon), Vercel Performance.

## Risks & Watchpoints

- **Troy Hunt - Broken Access Control:** R2 unlisted URLs are acceptable by decision, but database access must never rely on URL secrecy. Every tRPC procedure must filter by Clerk user id.
- **Troy Hunt - Secrets Management:** R2 keys, Neon URLs, Clerk secrets, and OpenRouter keys must remain server-only. No `NEXT_PUBLIC_` leakage.
- **Brandur Leach - Foreign State Mutation:** R2 upload plus DB insert is not atomic. The implementation needs idempotency and orphan cleanup, even for a personal app.
- **Brandur Leach - DateTime:** Use timezone-aware DB timestamps and compute day grouping deliberately. Food logs are time-sensitive and DST bugs will be confusing.
- **Matteo Collina - Resource Management:** Multipart/file upload handling must avoid unbounded memory. Start simple, but enforce size limits and consume/close request bodies correctly.
- **Kent C. Dodds - Server State:** Do not mirror all entries into manual local React state after moving to tRPC. Let query cache own server records.
- **Martin Fowler - Speculative Generality:** Do not build multi-user sharing, albums, tags, analytics, or background jobs as part of this migration.
- **Vercel Performance - Payloads:** Do not upload full iPhone originals blindly. Measure file sizes and use bounded image dimensions before storage and AI calls.

## Verdict

Use the full council stack now because this is no longer a purely local toy: Clerk provides identity, tRPC provides the typed API contract, Prisma/Neon provides durable ownership and query safety, and R2 provides cheap durable photo storage. The migration should be staged so current local photos are preserved, but after the migration IndexedDB must be treated only as temporary cache/migration input, never as source of truth.
